# Executing Plans - 执行计划

## 简介

Executing Plans skill用于在单独的会话中执行已编写的实施计划，带有审查检查点。

## 使用场景

- 有编写的实施计划要在单独会话中执行
- 需要批量执行任务并在批次之间进行审查
- 计划已经准备好，需要系统化实施

## 核心流程

### 步骤1：加载和审阅计划

1. 阅读计划文件
2. 批判性审阅 - 识别任何关于计划的问题或疑虑
3. 如有疑虑：在开始之前与你的伙伴提出
4. 如无疑虑：创建TodoWrite并继续

### 步骤2：执行批次

**默认：前3个任务**

对于每个任务：
1. 标记为in_progress
2. 确切遵循每个步骤（计划有小的步骤）
3. 按指定运行验证
4. 标记为completed

### 步骤3：报告

当批次完成时：
- 展示实现了什么
- 展示验证输出
- 说："准备好反馈。"

### 步骤4：继续

基于反馈：
- 如需要，应用更改
- 执行下一批次
- 重复直到完成

### 步骤5：完成开发

所有任务完成并验证后：
- 宣布："我正在使用finishing-a-development-branch skill来完成这项工作。"
- **必需的子技能：** 使用superpowers:finishing-a-development-branch
- 遵循该skill来验证测试、展示选项、执行选择

## 何时停止并寻求帮助

**立即停止执行当：**
- 在批次中遇到阻塞器（缺少依赖、测试失败、指令不清楚）
- 计划有阻止开始的严重缺口
- 你不理解指令
- 验证反复失败

**在猜测之前寻求澄清。**

## 何时返回早期步骤

**返回审阅（步骤1）当：**
- 伙伴基于你的反馈更新计划
- 基本方法需要重新思考

**不要强制通过阻塞器** - 停止并询问。

## 记住

- 批判性审阅计划先
- 确切遵循计划步骤
- 不要跳过验证
- 当计划说时参考skills
- 批次之间：只报告并等待
- 当阻塞时停止，不要猜测

## 示例工作流

```
你：我正在使用executing-plans skill来实施这个计划。

[阅读计划文件一次：docs/plans/feature-plan.md]
[批判性审阅...没有疑虑]
[创建TodoWrite包含所有任务]

Task 1: 设置数据库schema
[标记为in_progress]
[遵循步骤：创建users表...]
[运行验证：检查表存在]
[标记为completed]

Task 2: 实现注册端点
[标记为in_progress]
[遵循步骤：创建API...]
[运行验证：测试端点]
[标记为completed]

Task 3: 添加邮箱验证
[标记为in_progress]
[遵循步骤：实现验证逻辑...]
[运行验证：测试验证]
[标记为completed]

[批次完成]
报告：
- 实现了：数据库schema、注册端点、邮箱验证
- 验证：所有测试通过
- 准备好反馈。

伙伴：看起来不错，继续。

Task 4: 实现登录端点
[继续...]

[所有任务完成]
你：我正在使用finishing-a-development-branch skill来完成这项工作。
```
